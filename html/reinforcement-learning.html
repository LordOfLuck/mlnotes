Reinforcement learning

Based on Reinforcement learning: An Introduction and slides from Roman Barták. The notation has been adjusted to comply with the slides.

Supervised learning agent is explicitly told what to do. Receives input and required output.
Reinforcement learning agent only receives a numeric reward for his actions without being told what actions to take. The agent interacts with its environment, moves from state to state and based on itsa actions receives rewards from the environment.

Formally a reinforcement agent is placed in a markov decision process(MDP).
A MDP consists of the follwing:

  1. A set of states S\mathcal{S}S
  2. A set of available actions in each state As\mathcal{A}_sAs​ (all states can have identical action set)
  3. A transition function Pa(s,s′)=Pr(st+1=s′∣st=s,at=a)P_a(s, s') = Pr(s_{t+1} = s' | s_t = s, a_t = a)Pa​(s,s′)=Pr(st+1​=s′∣st​=s,at​=a)
  4. A reward function Ras,s′R_a{s, s'}Ra​s,s′ - immediate reward for doing action a in state sss and moving to state s′s's′

The transition probabilities between states are independent of previously visited states (markov property)
Reward function can depend only on state (we can transition between definitions: R(s)=EaEs′[Ras,s′]R(s) = \mathbb{E}_a\mathbb{E}_{s'}[R_a{s, s'}]R(s)=Ea​Es′​[Ra​s,s′])

The agent can have a

  1. deterministic policy π:S→As\pi : \mathcal{S} \rightarrow \mathcal{A_s}π:S→As​
  2. stochastic policy As∼π(.∣s)A_s \sim \pi (.|s)As​∼π(.∣s), where π(a∣s)=Pr(a∣s)\pi(a|s) = Pr(a|s)π(a∣s)=Pr(a∣s) is probability of taking action aaa in state sss.

The agent tries to maximize his utility - the expected discounted return (present value of future rewards).

Uπ(s)=Eπ[∑k=0∞γkRt+k,∣St=s],0≤γ≤1U_{\pi}(s) = \mathbb{E}_{\pi}[\sum_{k=0}^\infty \gamma^k R_{t+k}, |S_t = s], 0\leq \gamma \leq 1Uπ​(s)=Eπ​[k=0∑∞​γkRt+k​,∣St​=s],0≤γ≤1

We can divide agents into the following groups:

  1. Utility-based agent learns the values of UπU_{\pi}Uπ​ and selects a policy that is an improvement over the current one.
  2. Q-learning agent - learns action-value function - expected utility of taking a given action in a given state (this update is only for evaluation of policy!)

  3. Reflex agent learns a policy that maps directly from states to actions

We can divide types of learning:

  1. Passive - learn values UUU for fixed policy π\piπ
    • Direct utility estimation - collect experience by applying π\piπ, keep running average return U^(s)\hat{U}(s)U^(s) for each state. Supervised learning with (state, utility) pairs . Inefficient - it assumes independence between states, but we know the bellman equation which could help us shrink the hypothesis space.
    • Adaptive dynamic programming(ADP) - Instead of UUU, learn the transition probabilities (as frequencies) and rewards. Eg. repeatedly iterate over all states and in each state sample a reward and a random action and observe result state. Calculate UπU_{\pi}Uπ​ dynamically by backwards induction based on the estimates and chosen policy π\piπ or use policy evaluation.
    • temporal difference - Estimate UπU_{\pi}Uπ​ directly, but enforce consistency according to bellman equation - using the temporal difference update (smarter than direct utility estimation)
  2. Active - learn how to act optimally in an environment - must iteratively estimate the value function for current policy and update his policy to improve his expected gains. It involves exploration.
    • policy iteration, SARSA, Q-learning

Policy iteration - Iteratively evaluate UπU_{\pi}Uπ​ and improve π\piπ by selecting greedy actions w.r.t. current estimate of UUU. Thanks to policy improvement theorem this algorithm is guaranteed to converge, but we need accurate estimate of the environment transitions, which can easily be intractable.

  • for each epoch:
    • init sss

    • repeat:

      • Q(a,s)←Q(a,s)+α(R(s)+γQ(s′,a′)−Q(a,s)])Q(a, s) \leftarrow Q(a, s) + \alpha(R(s) + \gamma Q(s', a') - Q(a, s)])Q(a,s)←Q(a,s)+α(R(s)+γQ(s′,a′)−Q(a,s)])

      The algorithm is on-policy meaning that the agent behaves according to some policy (eg. ϵ\epsilonϵ-greedy) and updates its estimates to improve the same policy.

Q-learning - explore to search new states - with small proba ϵ\epsilonϵ , select any action with equal proba. Otherwise exploit (greedy (best) action in given state according to current estimate of QQQ in state sss)

  • for each epoch:
    • init sss
    • repeat:
      • a←EpsilonGreedy(Q,ϵ)a \leftarrow EpsilonGreedy(Q, \epsilon)a←EpsilonGreedy(Q,ϵ), observe R,s′R, s'R,s′
      • Q(a,s)←Q(a,s)+α(R(s)+γmax⁡aQ(s′,a)−Q(a,s)])Q(a, s) \leftarrow Q(a, s) + \alpha(R(s) + \gamma \max_{a} Q(s', a) - Q(a, s)])Q(a,s)←Q(a,s)+α(R(s)+γmaxa​Q(s′,a)−Q(a,s)])

      The update rule equality holds if the QQQ values are optimal (bellman)
      The algorithm is off-policy - the agent behaves according to ϵ\epsilonϵ-greedy policy, but makes updates to improve fully greedy policy (hard max in the update) so it improves different policy than the one that was used to collect the experience.
      Q-learning can learn an optimal policy even when a random policy is followed during training - more data efficient, easy to reuse experience, but may converge slower.

Exploration
Choosing optimal actions(exploitation) w.r.t. the current estimate of the environment may not be optimal w.r.t the true environment. Exploration means trying immediately suboptimal actions in order to discover states that are potentially more beneficial in the long-term. Exploration improves the model. The options are:

  1. ϵ\epsilonϵ-greedy - be greedy with proba (1−ϵ)(1-\epsilon)(1−ϵ), else explore uniformly
  2. Give large weight to actions with large variance estimate and avoid low variance low utility actions(actions that we know are bad)
    • eg. initialize UUU high for all states - optimistically biased (curious) agent. The agent will be mostly disappointed , but will have a motivation to discover.
